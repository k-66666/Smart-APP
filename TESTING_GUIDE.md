# 智能防霉柜系统测试指南

## 概述

本文档提供STM32智能防霉柜与Android应用的完整测试流程，确保系统协调工作。

## 测试环境准备

### 硬件要求

1. **STM32防霉柜系统**：
   - STM32F103C8主控板
   - ESP8266 WiFi模块
   - DHT11温湿度传感器
   - TVOC-301空气质量传感器
   - 继电器模块（风扇、紫外线、照明）
   - OLED显示屏
   - 门磁传感器
   - 光敏传感器

2. **Android设备**：
   - Android 5.0 (API 21) 或更高版本
   - 支持WiFi连接
   - 建议使用Android 8.0+以获得最佳体验

3. **网络环境**：
   - WiFi路由器
   - STM32和Android设备需在同一局域网

### 软件准备

1. **STM32端**：
   - 按照 `STM32_INTEGRATION_GUIDE.md` 集成命令接收功能
   - 配置WiFi SSID和密码
   - 烧录固件到STM32

2. **Android端**：
   - 编译并安装APK
   - 或使用模拟模式进行初步测试

## 测试流程

### 阶段1: 模拟模式测试（无需硬件）

#### 目的
验证Android应用的基本功能，无需实际硬件。

#### 步骤

1. **启动应用**
   ```
   打开MySmart应用
   ```

2. **进入设备管理**
   ```
   点击底部导航栏的"设备"图标
   ```

3. **添加模拟设备**
   ```
   点击"添加设备"按钮
   手动输入设备信息：
   - 设备名称: 模拟防霉柜
   - 设备ID: demo
   - 连接方式: WiFi
   ```

4. **连接模拟设备**
   ```
   点击"连接"按钮
   应该看到连接成功提示
   ```

5. **观察数据显示**
   ```
   返回"监控"页面
   应该看到模拟的传感器数据：
   - 温度: 20-30°C 随机变化
   - 湿度: 40-60% 随机变化
   - TVOC: 30-80 μg/m³
   - CO2: 400-1000 ppm
   数据每2秒更新一次
   ```

6. **测试控制功能**
   ```
   在"监控"页面点击控制按钮
   虽然没有实际硬件，但应该能看到：
   - 按钮状态变化
   - 命令发送成功提示
   ```

#### 预期结果
- ✅ 应用正常启动
- ✅ 可以添加和连接模拟设备
- ✅ 数据实时更新显示
- ✅ 控制按钮响应正常
- ✅ 无崩溃或错误

### 阶段2: STM32数据接收测试

#### 目的
验证Android应用能够正确接收和解析STM32发送的数据。

#### 前置条件
- STM32已烧录固件
- WiFi模块已配置并连接到路由器
- STM32正常运行并发送数据

#### 步骤

1. **查找STM32设备IP**
   ```
   方法1: 在路由器管理页面查看已连接设备
   方法2: 使用网络扫描工具（如Fing）
   方法3: STM32串口输出IP地址（如果有实现）
   ```

2. **在Android应用中扫描设备**
   ```
   打开"设备管理"页面
   点击"扫描设备"
   等待扫描完成
   应该能看到STM32设备（IP地址）
   ```

3. **连接STM32设备**
   ```
   点击扫描到的设备
   点击"连接"按钮
   等待连接建立（约3-5秒）
   ```

4. **验证数据接收**
   ```
   返回"监控"页面
   观察传感器数据：
   - 温度应该显示实际环境温度
   - 湿度应该显示实际环境湿度
   - TVOC和CO2应该显示传感器读数
   - 数据每5秒更新一次（与STM32发送间隔一致）
   ```

5. **验证设备状态同步**
   ```
   观察设备状态指示：
   - 风扇状态（开/关）
   - 紫外线状态（开/关）
   - 照明状态（开/关）
   - 报警状态（正常/报警）
   
   在STM32上按按键改变设备状态
   Android应该在5秒内同步显示新状态
   ```

#### 预期结果
- ✅ 能够扫描到STM32设备
- ✅ 连接成功
- ✅ 接收到真实的传感器数据
- ✅ 数据格式正确，无解析错误
- ✅ 设备状态正确同步
- ✅ 数据更新及时（5秒间隔）

#### 故障排查

**问题**: 扫描不到设备
```
检查项：
1. STM32和Android是否在同一WiFi网络
2. WiFi模块是否正常工作（查看OLED显示）
3. 路由器是否启用了AP隔离
4. 防火墙设置
```

**问题**: 连接失败
```
检查项：
1. 端口号是否正确（默认8080）
2. STM32是否已建立TCP服务器
3. 使用网络调试工具测试连接
```

**问题**: 数据解析失败
```
检查项：
1. 使用Wireshark抓包查看实际数据格式
2. 检查JSON格式是否正确
3. 查看Android Logcat日志
```

### 阶段3: Android控制STM32测试

#### 目的
验证Android应用能够成功控制STM32设备。

#### 前置条件
- 已完成阶段2测试
- STM32已集成命令接收功能（app_wifi_command.c）
- 设备处于连接状态

#### 测试用例

##### 测试用例3.1: 风扇控制

**步骤**：
1. 在"监控"页面找到风扇控制区域
2. 点击"开启"按钮
3. 观察STM32风扇是否启动
4. 点击"关闭"按钮
5. 观察STM32风扇是否停止
6. 点击"自动"按钮
7. 改变环境湿度，观察风扇是否自动控制

**预期结果**：
- ✅ 点击"开启"后，风扇立即启动
- ✅ 点击"关闭"后，风扇立即停止
- ✅ 自动模式下，湿度>65%时风扇自动开启
- ✅ 自动模式下，湿度<50%时风扇自动关闭
- ✅ Android界面状态与实际硬件状态一致

##### 测试用例3.2: 紫外线灯控制

**步骤**：
1. 确保柜门关闭
2. 点击紫外线"开启"按钮
3. 观察紫外线灯是否点亮
4. 打开柜门
5. 观察紫外线灯是否自动关闭（安全保护）
6. 关闭柜门
7. 点击"自动"按钮

**预期结果**：
- ✅ 门关闭时，可以手动开启紫外线
- ✅ 门打开时，紫外线自动关闭（安全锁定）
- ✅ 门打开时，无法通过Android开启紫外线
- ✅ 自动模式下，满足条件时自动启动消毒
- ✅ 消毒15分钟后自动关闭并报警

##### 测试用例3.3: 照明灯控制

**步骤**：
1. 点击照明"开启"按钮
2. 观察照明灯是否点亮
3. 点击"关闭"按钮
4. 观察照明灯是否熄灭
5. 点击"自动"按钮
6. 在暗环境下打开柜门
7. 观察照明灯是否自动点亮

**预期结果**：
- ✅ 手动控制响应及时
- ✅ 自动模式下，门开+暗环境时自动开灯
- ✅ 门关闭后延迟2秒关灯

##### 测试用例3.4: 多设备同时控制

**步骤**：
1. 同时开启风扇和照明
2. 观察两个设备是否都响应
3. 切换到不同的控制模式组合

**预期结果**：
- ✅ 多个设备可以独立控制
- ✅ 互不干扰

#### 故障排查

**问题**: 控制命令无响应
```
检查项：
1. STM32是否已集成app_wifi_command.c
2. 主循环是否调用APP_WiFi_ReceiveCommand()
3. 使用串口监控是否收到命令
4. 检查JSON格式是否正确
```

**问题**: 紫外线无法开启
```
检查项：
1. 柜门是否关闭
2. uv_locked标志状态
3. 是否处于消毒过程中
```

### 阶段4: 报警功能测试

#### 目的
验证报警系统的正确性。

#### 测试用例

##### 测试用例4.1: 湿度报警

**步骤**：
1. 人为增加环境湿度（如使用加湿器）
2. 等待湿度超过75%
3. 观察Android通知

**预期结果**：
- ✅ 湿度>75%时，STM32蜂鸣器响起
- ✅ Android收到报警通知
- ✅ 通知内容显示"湿度过高: XX%"
- ✅ 风扇自动开启除湿

##### 测试用例4.2: 消毒完成提示

**步骤**：
1. 手动启动紫外线消毒
2. 等待15分钟（或修改代码缩短时间测试）
3. 观察提示

**预期结果**：
- ✅ 消毒完成后，STM32蜂鸣器响起
- ✅ Android收到通知"紫外线消毒已完成"
- ✅ 紫外线灯自动关闭

##### 测试用例4.3: 空气质量报警

**步骤**：
1. 模拟TVOC或CO2浓度过高
2. 观察报警

**预期结果**：
- ✅ TVOC>5500时触发报警
- ✅ CO2>2000时触发报警
- ✅ Android显示相应通知

## 性能基准

### 响应时间

| 操作 | 预期时间 | 可接受范围 |
|------|---------|-----------|
| 设备扫描 | 5秒 | 3-10秒 |
| 建立连接 | 3秒 | 2-5秒 |
| 控制命令响应 | 500ms | <1秒 |
| 数据更新 | 5秒 | 5-6秒 |

### 数据准确性

| 传感器 | 精度要求 |
|--------|---------|
| 温度 | ±0.5°C |
| 湿度 | ±2% |
| TVOC | ±10% |
| CO2 | ±50ppm |
