# MySmart - 智能防霉柜监控系统

## 项目简介

MySmart是一个完整的智能防霉柜监控系统，由STM32硬件端和Android应用端组成。系统通过WiFi实现双向通信，提供实时环境监测、智能控制和数据记录功能。

### 主要功能

#### 🌡️ 环境监测
- **温湿度监测**: DHT11传感器，精度±0.5°C / ±2%
- **空气质量监测**: TVOC-301传感器，检测TVOC和CO2浓度
- **光线检测**: 自动感应环境光线
- **门磁检测**: 实时监控柜门开关状态

#### 🎛️ 智能控制
- **风扇除湿**: 
  - 自动模式：湿度>65%开启，<50%关闭
  - 手动模式：远程开关控制
- **紫外线消毒**:
  - 定时消毒：每天凌晨2点自动启动
  - 智能消毒：风扇运行30分钟+湿度>65%触发
  - 手动消毒：远程启动15分钟消毒
  - 安全保护：门开时自动锁定
- **智能照明**:
  - 自动模式：门开+暗环境自动开灯
  - 手动模式：远程开关控制
  - 延时关闭：门关后2秒自动熄灭

#### 📊 数据管理
- 实时数据显示
- 历史数据记录
- 数据趋势图表
- 数据导出功能

#### ⚠️ 报警系统
- 湿度过高报警（>75%）
- 空气质量报警（TVOC/CO2超标）
- 消毒完成提示
- 推送通知提醒

## 技术栈

### Android端
- **语言**: Java
- **最低版本**: Android 5.0 (API 21)
- **目标版本**: Android 14 (API 34)
- **架构**: MVVM
- **主要库**:
  - Room: 本地数据库
  - LiveData: 响应式数据
  - MPAndroidChart: 数据图表
  - Material Design: UI组件

### STM32端
- **芯片**: STM32F103C8
- **开发环境**: Keil MDK
- **标准库**: STM32F10x_StdPeriph_Lib
- **通信**: USART + ESP8266 WiFi
- **传感器**: DHT11, TVOC-301
- **显示**: OLED (I2C)

## 通信协议

### 数据格式
使用JSON格式进行数据交换，通过TCP/IP协议传输。

#### 上行数据（STM32 → Android）
```json
{
  "temp": 25.5,
  "humi": 60.3,
  "tvoc": 150,
  "co2": 450,
  "fan": 1,
  "uv": 0,
  "light": 1,
  "alarm": 0
}
```

#### 下行控制（Android → STM32）
```json
{
  "cmd": "control",
  "fan": 1,
  "uv": 0,
  "light": 2
}
```

详细协议说明请参考 [PROTOCOL.md](PROTOCOL.md)

## 快速开始

### Android应用

#### 环境要求
- Android Studio 2023.1 或更高版本
- JDK 17
- Android SDK 34

#### 编译步骤
```bash
# 克隆项目
git clone <repository-url>
cd MySmart

# 打开Android Studio
# File -> Open -> 选择项目目录

# 同步Gradle
# 点击 "Sync Project with Gradle Files"

# 编译运行
# Run -> Run 'app'
```

#### 安装APK
```bash
# 生成Release版本
./gradlew assembleRelease

# 安装到设备
adb install app/release/app-release.apk
```

### STM32固件

#### 环境要求
- Keil MDK 5.x
- STM32F10x标准库
- ST-Link调试器

#### 编译步骤
1. 打开Keil项目文件 `User/ZNYG.uvprojx`
2. 配置WiFi参数（`Drivers/CONFIG/sys_config.h`）
3. 编译项目（F7）
4. 下载到STM32（F8）

详细集成说明请参考 [STM32_INTEGRATION_GUIDE.md](../Stm32-ZNYG/STM32_INTEGRATION_GUIDE.md)

## 使用指南

### 1. 硬件准备
- 组装STM32防霉柜系统
- 连接所有传感器和执行器
- 配置WiFi模块

### 2. 软件配置

#### STM32端
```c
// 修改 sys_config.h
#define WIFI_SSID         "你的WiFi名称"
#define WIFI_PASSWORD     "你的WiFi密码"
#define WIFI_TCP_PORT     8080
```

#### Android端
```java
// 默认配置在 DeviceConfig.java
// 通常无需修改，与STM32保持一致
```

### 3. 连接设备

1. 启动STM32系统
2. 打开Android应用
3. 进入"设备管理"页面
4. 点击"扫描设备"
5. 选择发现的设备并连接

### 4. 开始使用

- **监控页面**: 查看实时数据和控制设备
- **数据页面**: 查看历史数据和趋势
- **设备页面**: 管理连接的设备
- **设置页面**: 配置报警阈值等参数

## 配置说明

### 湿度控制阈值
```java
// DeviceConfig.java
HUMIDITY_HIGH_THRESHOLD = 65.0f;  // 风扇开启阈值
HUMIDITY_LOW_THRESHOLD = 50.0f;   // 风扇关闭阈值
HUMIDITY_ALARM_THRESHOLD = 75.0f; // 报警阈值
```

### 消毒参数
```java
UV_DISINFECT_DURATION_MINUTES = 15;  // 消毒时长
FAN_RUN_TIME_BEFORE_UV_MINUTES = 30; // 风扇预运行时间
```

## 故障排查

### 常见问题

#### 1. 无法连接设备
**症状**: 扫描不到设备或连接失败

**解决方法**:
- 检查WiFi连接
- 确认在同一局域网
- 检查端口号（默认8080）
- 查看路由器是否启用AP隔离

#### 2. 数据不更新
**症状**: 连接成功但数据不变化

**解决方法**:
- 检查STM32是否正常运行
- 查看串口输出
- 使用网络抓包工具检查数据包

#### 3. 控制命令无响应
**症状**: 点击控制按钮无效果

**解决方法**:
- 确认STM32已集成命令接收功能
- 检查JSON格式
- 查看Android Logcat日志

更多问题请参考 [TESTING_GUIDE.md](TESTING_GUIDE.md) 的故障排查章节。
